## 1. Тема и целевая аудитория <a name="1"></a>

**Booking.com** — лидер мирового рынка онлайн-бронирования путешествий. Сервис объединяет миллионы объектов размещения (от частных апартаментов до пятизвездочных отелей) с арендаторами по всему миру.

### MVP
- Поиск объектов с фильтрацией (даты, цена, рейтинг, удобства).
- Страница объявления с фотографиями и описанием.
- Система бронирования (транзакции, управление статусами).
- Профили арендатора и арендодателя.
- Система двусторонних отзывов и рейтингов.
- Статус доступности объекта в реальном времени.

### Целевая аудитория
- **Количество посещений в месяц:** ~560 млн пользователей [^1]
- **DAU (Daily Active Users):** ~17 млн активных пользователей [^2]
- **Объекты:** ~28 млн объявлений [^8]
- **Пользователи:** ~80 млн(на основе данных по Trip Adviser)[^5]

## 2. Расчет нагрузки <a name="2"></a>

При показателях MAU ~560 млн и DAU ~17 млн, мы можем вычислить пиковую нагрузку. Данные по количеству зарегистрированных пользователей не были найедны, поэтому возьмем, как аналог, данные Tripadvisor - ~80 млн[^5]
Региональное распределение трафика [3]:

| Регион | Доля аудитории | MAU | DAU |
| :--- | :--- | :--- | :--- |
| США | 10,13% | 56.7 млн | 1.7 млн |
| Великобритания | 7,67% | 43 млн | 1.3 млн |
| Германия | 6,5% | 36.4 млн | 1.1 млн |
| Италия | 6,14%  | 34.4 млн | 1.04 млн |
| Другие | 63,44% | 355.3 млн | 10.8 млн |
* **Средний размер хранилища пользователя:**

| Данные           | Кол-во на одного пользователя | Размер единицы | Общий размер |
| ---------------- | ----------------------------- | -------------- | ------------ |
| Профиль          | 1                             | 1 МБ           | 1 МБ         |
| Текстовые отзывы | 4                             | 2 КБ           | 8 КБ        |
| Фото в отзывах   | 2                             | 500 КБ         | 1 МБ         |

*Профиль*: фото профиля (500 КБ), обложка профиля (500 КБ), остальная информация (имя, дата присоединения, избранное и т.п.) (10 КБ).

*Текстовые отзывы*: на данный момент на сайте опубликовано около 300 млн отзывов[^4]. Отзывы могут оставлять только зарегистрированные пользователи. Итак, в среднем 300млн / 80 млн ~ 4 отзыва на 1 пользователя.

*Фото в отзывах*(данные взяты по TripAdvisor): путешественники опубликовали более 160 млн таких фото[^6]. Итак, в среднем 160 млн / 80 млн = 2 фото на 1 пользователя.

Таким образом, данные, связанные с 1 пользователем, занимают в среднем **2 МБ** хранилища.

#### Среднее значение действий пользователя в день:

| Действие                          | Среднее количество в день на пользователя |
|-----------------------------------|-------------------------------------------|
| Поиск по параметрам               | 2                                         |
| Получение объявления              | 6                                         |
| Просмотр отзывов                  | 4                                         |
| Загрузка/изменение объявления     | 0.1                                       |
| Регистрация/Авторизация           | 0.5                                       |
| Бронирование                      | 0.2                                       |
| Публикация отзывов                | 0,001                                     |

*Публикация отзывов*: за 2022 год было опубликовано 30,2 млн отзывов[^7]. 30 млн / 80 млн / 12 месяцев / 30 дней ~ 0,001 отзыва в день в среднем на 1 зарегистрированного пользователя.


### Технические метрики

* **Общий объем хранимых данных:**

| Данные       | Размер |
| ------------ | ------ |
| Бронирование      | 44 ТБ  |
| Отзывы | 74 ТБ |
| Объявления       | 147 ТБ  |
| профиль      | 160 ТБ  |
*Объявления*: для каждого объявления хранятся общие данные (название, описание, адрес, рейтинг, категории и т.п.) (~ 0,5 МБ), фотографии (около 10 штук по 500 КБ каждая). Общее число объектов[^8] - 28 млн. Тогда (0,5 МБ + 10 * 0,5 МБ) * 28 млн = 147 ТБ.

*Профили*: 2 МБ * 80 млн (зарегистрированных пользователей[^5]) = 160 ТБ.

*Отзывы*: общее число отзывов[^4] - 300 млн,  общее число фото[^5] - 160 млн. Тогда 0,002 МБ * 300млн + 0,5 МБ * 160 млн = 2 ТБ + 80 ТБ = 74 ТБ.

*бронирование*: примерно 0.5 КБ на одно бронирование, количество броней за последние 5 лет в среднем 4450 млн. Тогда 4475 млн. * 0.5 КБ = 2.3 ТБ

Общий объем хранилища составляет **383 ТБ**.

#### Сетевой трафик и RPS

Используя. "инструменты разработчика" в браузере мы можем примерно рассчитать размер .html, .js и .css файлов для
отрисовки каждой из страниц и количество запросов для их получения, в качестве данных возьмём средний результат
знакомых:

| Страница                         | Размер в Кб | Кол-во запросов |
|----------------------------------|-------------|-----------------|
| Поиск по параметрам              | 613         | 260             |
| Получение объявления(с отзывами) | 1000        | 560             |
| Регистрация/Авторизация          | 455         | 40              |
| Бронирование                     | 770         | 190             |

Получаем средний размер трафика и запросов для **статики на одного пользователя в день**:

```azure
Трафик: 613 * 2 + 1000 * (6 + 1) + 455 + 770 = 9.5 Мб
Кол-во
запросов: 260 * 2 + 560 * (6 + 1) + 40 + 190 = 4670
```

Тогда учитывая `17M MAU`, получаем:

```azure
Трафик: 9.5 Мб * 17М / 86 400 = 1.98 Гб/c
RPS: 4670 * 18М / 86 400 = 973 000
```
